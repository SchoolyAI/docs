## Setting Up Your Development Environment

Follow these steps to set up your local development environment for contributing to SchoolyAI:

### Step 1: Clone the Repository

```bash
git clone https://github.com/SchoolyAI/scorton-doc.git
cd scorton-doc
```

### Step 2: Install Dependencies

<CodeGroup>

```bash npm
npm install
```

```bash yarn
yarn install
```

</CodeGroup>

### Step 3: Set Up Environment Variables

Create a `.env.local` file in the root directory:

```bash
cp .env.example .env.local
```

Edit the `.env.local` file with your configuration:

```env
# API Configuration
NEXT_PUBLIC_API_URL=https://api.schoolyai.com/v1
NEXT_PUBLIC_APP_URL=http://localhost:3000

# Authentication
NEXTAUTH_SECRET=your-secret-key
NEXTAUTH_URL=http://localhost:3000

# Database (if using local development)
DATABASE_URL=your-database-url

# External Services
OPENAI_API_KEY=your-openai-key
STRIPE_SECRET_KEY=your-stripe-key
```

### Step 4: Run the Development Server

```bash
npm run dev
```

Your local development server will be available at `http://localhost:3000`.

### Custom Ports

By default, the development server uses port 3000. You can customize the port by using the `--port` flag:

```bash
npm run dev -- --port 3333
```

## Code Quality

### Linting and Formatting

We use ESLint and Prettier for code quality. The project includes pre-commit hooks that automatically format your code:

```bash
npm run lint:fix
```

### TypeScript

This project uses TypeScript for type safety. Run type checking:

```bash
npm run type-check
```

### Testing

Run the test suite:

```bash
npm run test
```

For watch mode during development:

```bash

```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port 3000 is already in use">
    The development server will automatically try the next available port. You can also manually specify a different port:
    ```bash
    npm run dev -- --port 3001
    ```
  </Accordion>

  <Accordion title="Database connection issues">
    Ensure your database is running and the connection string in `.env.local` is correct. For local development, you can use Docker:
    ```bash
    docker-compose up -d
    ```
  </Accordion>

  <Accordion title="API key issues">
    Make sure your API keys are properly set in the `.env.local` file. You can get test API keys from the SchoolyAI dashboard.
  </Accordion>
</AccordionGroup>
